Oversight
Portal

NADIR enables role-based, deterministic verification of integrity-sealed artifacts. Calibrated access without default exposure of proprietary internals.

Section A // Role-Based Access

Role: Regulated Operator
  • • Defines Policy ID & Scope
  • • Manages local key custody
  • • Responds to artifact requests
Access: Full Set (Policy Scoped)
Role: Authorized Institution
  • • Receives controlled artifacts
  • • Recomputes hashes & sigs
  • • Issues oversight determinations
Access: Controlled Review Tier
Role: Forensic Auditor
  • • Retrieves incident-scoped sets
  • • Independent verification
  • • Chain-of-custody production
Access: Mandate-Scoped
Note: Verification confirms integrity and provenance. It does NOT confirm decision correctness.

Section B // Five-Step Workflow

1. Retrieve
2. Canonicalize
3. Re-Hash
4. Validate Sig
5. Confirm Policy

Section C // Disclosure Tiers

Public / Redacted

Artifact summary, Policy ID, timestamp. No sensitive fields. Open access.

Controlled Review

Expanded fields, full signature, environment metadata. Requires executed NDA / review agreement.
