Integrations · Attachment patterns · Exports

Fits your stack — as a layer.

CEYO is designed to attach at the inference boundary and emit artifacts into storage, security, and governance systems. Integration preserves existing inference infrastructure and avoids default exposure of model internals.

Integration patterns

Common ways operators attach CEYO and route artifacts.

Boundary Gateway

Model gateway / proxy

Place CEYO between callers and the inference endpoint. Capture and seal artifacts while requests pass through unchanged (fail-open supported).

mode: "gateway" fail_open: true capture_policy: "POL-001@V1"
Boundary Sidecar

Sidecar (co-located)

Deploy alongside the inference service in containerized environments. Minimal pipeline disruption; designed for low overhead.

mode: "sidecar" intercept: "localhost" kms_signing: true
Boundary Wrapper

Service wrapper

Expose a compatible interface that calls inference internally and emits artifacts in parallel. Suitable when interface control is required.

mode: "wrapper" upstream: "inference" emit_artifact: "async"
Export Storage

Secure artifact storage

Write artifacts to an access-controlled repository with retrieval logging for chain-of-custody compatible workflows.

store: "object_storage" retention: "institution_policy" access_log: true
Export SIEM

Security tooling (SIEM/SOAR)

Emit artifact metadata and verification outcomes to security systems for monitoring, alerting, and incident workflows.

export: "siem" events: ["sealed","verify_fail","policy_mismatch"] severity_map: "operator_defined"
Export GRC

Governance & audit

Export evidence references for audit programs: policy versions, signatures, verification outcomes, and retrieval trails.

export: "grc" fields: ["policy_id","policy_version","digest","signature","result"] tier: "controlled_review"

Deployment modes

Common operating environments. Operators define custody, retention, and disclosure boundaries.

Cloud
  • Integrate with managed KMS/HSM for signing operations.
  • Artifacts stored in access-controlled object storage with audit logs.
  • Scale horizontally; determinism enforced by canonicalization rules.
On-prem / air-gapped
  • Sign with on-prem HSM/Vault; no external dependencies required.
  • Artifacts retained in institutional repositories and exported by policy.
  • Disclosure tiers aligned to internal handling requirements.
