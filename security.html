Security · Key custody · Failure modes

Your keys remain yours.

CEYO is designed as a non-custodial evidentiary layer: private signing keys remain in the operator environment (HSM/KMS/TEE). CEYO produces sealed artifacts for verification workflows under explicit policy scope.

Key custody model

Signing authority stays with the operator. CEYO requests sign operations without holding private key material.

Non-custodial signing
  • Private keys never reside in CEYO application memory as durable material.
  • Sign operations are delegated to operator-controlled KMS/HSM/Vault/TEE.
  • Artifacts record the public key reference or key identifier used.
What verification proves
  • Integrity Artifact has not been modified since sealing.
  • Provenance Signature validates under declared key reference.
  • Scope Policy ID/version/hash identify capture boundary.
Signing flow

The private key remains within the operator custody boundary. CEYO stores the signature and key reference in the artifact.

Failure modes

Default posture prioritizes inference continuity. Failure events are logged and become reviewable signals.

KMS unavailable

Inference continues (fail-open). Artifact is marked unsealed and a failure event is recorded for review.

Timeout / overload

Inference continues. CEYO records a seal failure event and preserves policy/version context for auditability.

Policy mismatch

Verification reports POLICY_MISMATCH when policy ID/version/hash does not align with expected policy document.
