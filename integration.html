OPERATIONAL INTEGRATION

NADIR is designed to integrate without requiring modification of core model weights or inference logic. It operates as an attachable integrity layer.

Model A

Sidecar

Operates as a parallel process alongside the inference service, observing events via local IPC or network tap.

Model B

Gateway

Integrates as a transparent proxy layer intercepting and sealing inference requests and responses.

Model C

API Wrapper

Encloses inference calls within a hardened wrapper to ensure sealing occurs at the execution boundary.

Policy-Scoped Capture

Artifact generation is governed by explicit policy. The deploying institution defines included fields, exclusion rules, and masking behavior. NADIR does not ingest model weights or training datasets by default.

The Artifact Lifecycle

  1. Capture: Policy-scoped fields are isolated during the inference event.
  2. Normalization: Context is structured into a stable, canonical schema.
  3. Sealing: Deterministic hashing and signing are applied via HSM.
  4. Persistence: Artifact is stored according to institutional retention posture.

Integration Position

NADIR is infrastructure that enables structured review. It does not intervene in decision-making or govern operational authority. Integration is incremental; oversight remains institutional.
