Roadmap

The objective is to move from concept to proof: a deterministic artifact pipeline, a verification harness, and a retention-ready record format before expanding scope.

Phase 1 — Minimal viable artifact

Deliverable: deterministic artifact generation + reproducibility tests under defined conditions.

Define capture boundary

Specify exactly what is captured at inference time and what is excluded to preserve confidentiality.

Canonicalize inputs

Normalize ordering, serialization formats, rounding policy, and metadata to reduce entropy.

Generate DWSH signature

Produce a tamper-evident fingerprint derived from the abstracted representation.

Verification harness

Prove repeatability under identical conditions; detect mismatches and tampering.

Phase 2 — Record format & custody posture

Deliverable: retention-ready schema + verification procedure suitable for review workflows.

  • Record schema: timestamps, provenance metadata, input hash, abstraction ID, signature, verification fields.
  • Retention posture: storage, access controls, and lifecycle alignment with evidentiary practices.
  • Controlled reporting: review outputs that preserve IP boundaries.

Phase 3 — Controlled expansion

Deliverable: broader coverage while maintaining determinism and conservative claims.

  • Model class extension: define capture/canonicalization by runtime and deployment mode.
  • Environment hardening: document determinism boundaries across hardware and distributed inference.
  • Independent validation protocol: verification steps that do not require weight disclosure.
