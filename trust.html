Verification Procedure

A Mechanical
Process

Verification confirms integrity and provenance under declared policy scope. It does not confirm correctness of model output. Evidence remains separate from decision authority.

Recompute Canonical bytes
A verifier canonicalizes the artifact using the same declared procedure, producing stable bytes for independent hashing.
Validate Hash + signature
Hash recomputation confirms tamper-evidence. Signature validation confirms origin authenticity under the declared suite.

Trust Properties

What Verification Proves

01 · Determinism Stable bytes
Canonicalization produces a stable byte representation so independent parties can recompute identical digests without interpretive variance.
02 · Tamper-Evidence Hash mismatch
Any alteration of canonical bytes changes the digest, providing mechanical tamper-evidence.
03 · Provenance Signature validity
Signature validation confirms origin authenticity under the declared suite and public key material.
04 · Scope Integrity Declared boundary
The artifact includes policy identifiers so verification is tied to declared capture scope, exclusions, masking, and tiering.
