Verification Surface

A Minimal
Verifier Contract

Verification should not require privileged access. A verifier needs: the artifact, the policy identifier/version, and the public verification material appropriate to the disclosure tier.

Inputs What verifiers need
  • Artifact payload (declared schema)
  • Policy identifier + version
  • Canonicalization reference (procedure)
  • Hash suite + signature suite identifiers
  • Public key / certificate chain (tier-dependent)
Outputs What verifiers assert
  • Canonical bytes match policy + procedure
  • Recomputed hash matches recorded hash
  • Signature validates against public material
  • Artifact integrity status (sealed/unsealed)
  • Policy scope provenance (versioned)

Disclosure
Verification confirms integrity and provenance under policy scope. It does not require default exposure of proprietary internals. Tiered disclosure determines what is visible to which verifier class.
Mechanical Verification

Proof Steps

01 · Parse Schema & policy
Confirm artifact schema version and embedded policy identifiers. Ensure the verifier uses the correct policy version and tier rules for the artifact under review.
02 · Canonicalize Deterministic bytes
Apply the canonicalization procedure to produce stable byte output. Determinism eliminates interpretive variance across platforms and implementations.
03 · Hash Recompute
Recompute the digest over canonical bytes using the declared hash suite. Compare to the recorded digest to confirm tamper-evidence.
04 · Verify Signature validation
Validate the signature against public verification material appropriate to the disclosure tier. This confirms origin authenticity and integrity.
05 · Assert Integrity status
Produce a verification statement: sealed/unsealed, policy version, suites used, and validation result. No claims about correctness or compliance outcomes.
06 · Fail-Open Continuity preserved
If sealing fails, inference proceeds uninterrupted. Failure is logged and the artifact is marked unsealed, enabling review without blocking operations.
