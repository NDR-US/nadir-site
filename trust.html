Trust Center

NADIR’s trust posture is defined by limited claims, verifiable artifacts, and non-custodial architecture. Public content describes principles only.

[POSTURE]

Trust through constraint

Non-custodial cryptography

Private signing keys remain under client control (HSM/KMS). NADIR does not hold or escrow keys.

Deterministic canonicalization

JSON artifacts are canonicalized using RFC 8785 prior to hashing to prevent cross-system divergence.

Fail-open logging

If capture fails, inference proceeds. Failures are logged. NADIR does not block inference pipelines.

Disclosure tiers

Artifact visibility is configurable. Verification can occur without default exposure of proprietary internals.
