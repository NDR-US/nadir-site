DECOUPLED VERIFICATION ARCHITECTURE

NADIR is designed as a verification substrate that operates beside AI inference systems without modifying model weights or core decision logic.

Attachment at the Boundary

Integration occurs at the inference boundary via sidecar, gateway, or API wrapper. This posture allows NADIR to observe defined events and produce deterministic artifacts without modifying model internals or mutating training pipelines.

// Technical Verification Protocol

1. CAPTURE: Policy-scoped context normalized into stable schema.

2. CANONICALIZE: RFC 8785 (JSON Canonicalization Scheme) applied.

3. HASH: SHA-256 integrity digest computed.

4. SEAL: ECDSA (P-256) digital signature applied via HSM/KMS.

Non-Custodial Key Architecture

Signing keys are generated and retained within client-controlled environments (HSM, KMS, or TEE). NADIR does not possess, transmit, or store private signing keys. Artifact authenticity remains anchored to the deploying institution.

Architectural Position

NADIR is a verification substrate. It does not validate output correctness or certify compliance. Verification confirms integrity and provenance under declared policy. It provides deterministic artifacts; institutions determine their meaning.
