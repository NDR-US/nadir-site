Platform · Components · Interfaces

Component architecture, not a monolith.

CEYO is a boundary-layer evidentiary system: policy scope defines what is recorded, canonicalization makes it deterministic, sealing makes tampering detectable, and verification makes integrity independently provable.

Design posture

Built for institutional review: deterministic outputs, explicit scope, non-custodial keys, and constrained disclosure.

  • Attach beside inference (sidecar / gateway / wrapper)
  • Record only policy-scoped context
  • Canonicalize deterministically (RFC 8785)
  • Seal with hash + signature under operator custody

Capture policy

Defines explicit capture scope and disclosure tiers before artifact generation begins.

  • Records policy identifier and version within each artifact.
  • Applies masking and exclusion before canonicalization and hashing.
  • Prevents implicit expansion beyond declared scope.
policy_id: "POL-001" policy_version: "V1" masking: ["pii.redact","secrets.hash"] tier: "controlled_review"
