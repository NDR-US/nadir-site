CEYO is designed as a weight-neutral integrity layer that attaches beside inference systems. It captures policy-scoped context, normalizes into canonical artifacts, canonicalizes deterministically (RFC 8785), and cryptographically seals records for later independent verification.

CANONICAL ARTIFACT GENERATION

Artifact Pipeline

CEYO operates at the inference boundary as a sidecar, gateway, or API wrapper. It observes defined inference events and records only what policy declares.

  1. Policy-scoped capture (include/exclude/mask before generation)
  2. Canonical schema normalization (deterministic structure)
  3. RFC 8785 canonicalization (stable byte output)
  4. Hash computation (tamper-evidence)
  5. Digital signature (origin authenticity)
  6. Retention + retrieval posture (operator governed)
Availability

In fail-open configuration, inference continues even if sealing fails. Failures are logged and artifacts are marked unsealed. Availability remains an operator decision.

01 / Attach Sidecar · gateway · wrapper
CEYO attaches at the inference boundary to observe defined events without modifying model weights or decision logic. The inference engine remains unchanged.
02 / Policy Define before collect
Capture boundaries are explicit and versioned. Inclusion, exclusion, masking, and disclosure tiers are defined prior to artifact generation. Policy identifiers and versions are recorded within the artifact.
03 / RFC 8785 Deterministic bytes
JSON is canonicalized using RFC 8785 (JSON Canonicalization Scheme) so logically equivalent structures produce identical byte representations across environments, enabling independent recomputation.
04 / Crypto Hash + signature
After canonicalization, hashing and digital signatures seal artifacts. Reference suite: SHA-256 and ECDSA (P-256), configurable by deployment requirements.
