PLATFORM & ARCHITECTURE

Decoupled verification
architecture.

CEYO operates beside inference systems as an evidentiary substrate. It does not modify model weights or decision logic. It generates deterministic, policy-scoped artifacts — canonicalized and sealed for later institutional verification under constrained disclosure.

01 / ATTACH

Inference boundary attachment

Deploy as a sidecar, gateway, or wrapper at the inference boundary. The inference engine remains unchanged. The evidentiary layer operates externally to preserve architectural neutrality.

02 / CAPTURE

Policy-scoped fields only

Capture is explicitly defined before operation. Policy determines inclusion, exclusion, and masking rules. Fields not declared in scope are dropped prior to artifact generation.

03 / CANONICALIZE

Deterministic canonicalization

Normalize captured context into a stable artifact schema. Apply RFC 8785 JSON Canonicalization (JCS) to produce stable bytes across environments. Canonicalization occurs after policy-defined masking and exclusion.

04 / SEAL

Hash + signature

Compute a cryptographic digest over canonical bytes, then apply a digital signature. Reference suite: SHA-256 + ECDSA P-256, with suite selection configurable to institutional requirements. Private keys remain operator-controlled (HSM/KMS/TEE).

05 / DISCLOSE

Tiered disclosure model

Public / Redacted and Controlled Review tiers are defined before artifact generation to avoid post hoc ambiguity. Verification can confirm integrity under declared scope without default exposure of proprietary model internals.

06 / VERIFY

Independent verification workflow

Recompute canonicalization and hash deterministically, validate the signature against the public key reference, and confirm policy identifier/version alignment. Verification confirms integrity and provenance under declared policy scope — not correctness of model output.

OPERATIONAL CONSTRAINTS
FAIL-OPEN BY DEFAULT
FAIL-OPEN CONTINUITY

Inference must proceed if sealing fails. Exceptions are logged and become reviewable integrity events.

DETERMINISM PRESERVED

Same input + same policy version yields identical canonical bytes and digest across instances and environments.

NON-CUSTODIAL KEYS

Private signing keys remain under operator control. CEYO does not store or transmit private signing material.

SCOPE IS DECLARED

Artifacts embed policy identifiers/versions to make capture boundaries explicit and reviewable.
