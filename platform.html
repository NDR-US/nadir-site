Architecture and signature model

NADIR is positioned as an inference-boundary overlay that produces deterministic audit artifacts. The DWSH signature is designed to be verifiable under constrained disclosure while preserving proprietary boundaries.

Integration & use cases Briefing suite

System configuration

A structured pipeline designed for audit workflows: capture state → canonicalize → abstract → sign → store artifact.

Input state capture

Receives structured inference inputs; generates an input baseline hash at time t₀.

Deterministic canonicalization

Normalizes ordering, serialization formats, rounding policy, and metadata to reduce accidental entropy.

Decision-state abstraction

Transforms internal inference vectors into compliance-safe representations under a defined determinism boundary.

DWSH signature

Generates a tamper-evident fingerprint derived from the abstracted representation for later validation.

Functional constraints

These constraints are designed to keep the system legally defensible and adoption-ready.

  • Reproducible under identical conditions (documented determinism boundary).
  • Does not disclose model weights or source code.
  • Supports evidentiary preservation workflows (integrity, provenance, chain-of-custody posture).
  • Does not claim decision correctness.

What this is not

Clear boundaries prevent misinterpretation and over-claiming.

  • Not a safety system by itself.
  • Not an “explainability” product requiring internal disclosure.
  • Not a replacement for policy gating, human review, or authorization controls.
  • Not a claim of cross-hardware determinism unless explicitly proven and documented.

Determinism boundary (required)

Determinism is engineered, not assumed. The artifact pipeline requires documented capture scope and execution conditions.

Category What is controlled Why it matters
Input canonicalization Ordering, normalization, serialization, rounding policy Prevents accidental entropy that breaks reproducibility
Runtime conditions Hardware class, kernels, precision mode, randomness control Inference can diverge across environments without constraints
Instrumentation Stable hooks, consistent capture points Ensures the same representation is captured each time
