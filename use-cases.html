Integration • Runtime hooks • Operations

Designed to integrate beside inference

NADIR is structured as an attachable layer: capture, artifact emission, and verification—without becoming the model.

Deployment patterns

  • Sidecar to inference service (preferred)
  • Gateway mode (policy + capture at the edge)
  • Embedded SDK (when sidecar is not feasible)
  • Offline verifier (auditor workstation validation)

Operational controls

  • Retention windows and redaction profiles
  • Incident flags and escalation triggers
  • Audit sampling for high-volume systems
  • Role-based access and controlled review

Performance posture

Designed to minimize inference impact; capture scope is configurable by policy level.

Chain-of-custody ready

Artifacts are emitted with integrity fields for later verification and review board workflows.

Disclosure boundaries

Auditors verify artifacts without requiring model weight disclosure or proprietary code exposure.

Example outcomes

Post-incident replay

Reconstruct a disputed decision using the stored evidence package and integrity chain.

Oversight review

Provide reviewers a bounded, reproducible record aligned to disclosure level and retention policy.
