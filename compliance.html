Compliance · Evidence outputs · Control alignment

Evidence outputs mapped to controls.

CEYO provides deterministic, integrity-sealed artifacts that support verification and audit workflows under explicit policy scope. This page is descriptive. It does not constitute certification or legal advice.

Plain-language summary
  • Operators define capture scope and disclosure tiers before artifacts are generated.
  • Artifacts are canonicalized and sealed to make tampering detectable and provenance verifiable.
  • Auditors verify integrity mechanically (canonicalize → hash → signature → policy alignment).

Compliance matrix (descriptive)

The mappings below describe functional correspondence between CEYO artifacts and common integrity-oriented control expectations. Suitability and sufficiency are determined by the deploying institution and relevant authority.

Control expectation CEYO output Verification method Notes / boundary
Record-keeping Canonical artifact with policy ID/version and timestamp. Recompute canonical bytes and hash; validate signature. Describes integrity of the record, not correctness of the decision.
Integrity Digest + signature over canonical payload. PASS / FAIL outcome from deterministic procedure. Tamper-evidence applies from seal time forward.
Provenance Public key reference / KMS key ID and signature metadata. Signature validation using referenced public key chain. Key custody is operator responsibility (non-custodial model).
Scope discipline Policy identifier and version embedded in artifact (policy hash where used). Policy alignment check: expected ID/version/hash vs artifact. Scope is defined before generation; no implied expansion.
Constrained disclosure Disclosure tier designation (public/redacted, controlled review). Verify within tier boundaries; no need for proprietary internals. Tier sufficiency for proceedings is authority-determined.
Auditability Stable schema + canonicalization version recorded for repeatable validation. Re-run verification across environments with same inputs and policy. Does not guarantee inference reproducibility beyond scope.
Availability Fail-open posture (seal failures logged/flagged). Operational logs + artifact sealing status review. Availability posture is operator-configured.
Boundary declaration

CEYO produces integrity-sealed records. It does not certify compliance, legality, or decision correctness. Verification confirms integrity and provenance under policy scope — not correctness of model output.
