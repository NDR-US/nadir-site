How it works · Deterministic mechanics · Independent verification

Evidence generation is a mechanical chain.

CEYO attaches beside inference systems and produces an audit artifact through a fixed sequence: capture → canonicalize → seal → disclose → verify. Verification confirms integrity and provenance under policy scope — not correctness of the model output.

View the steps→ Verify in 60 seconds→

Five steps

Each step is simple in plain language. Technical detail is available on demand.

01 · Attach
Deploy as a layer

Sidecar, gateway, or wrapper at the inference boundary.

02 · Capture
Define scope first

Policy-scoped fields only. Exclusions and masking recorded.

03 · Canonicalize
Normalize deterministically

Stable schema + RFC 8785 JSON canonicalization.

04 · Seal
Make tampering detectable

Hash + digital signature under operator key custody.

05 · Verify
Independent validation

Recompute canonicalization + validate signature + policy alignment.

01 · Attach

CEYO runs beside an inference endpoint. It records evidence without changing model weights or decision logic.

Attachment is deployed as a boundary-layer pattern (sidecar, gateway proxy, or wrapper). The inference call can pass through unchanged. Fail-open configuration avoids introducing a default operational dependency.

Verify in 60 seconds

This is a demonstrator UI. It validates format discipline and shows the verification steps. Cryptographic verification is performed in the verifier toolkit.

Verification walkthrough
Status · waiting
1) Canonicalize under the recorded scheme RFC 8785 JSON canonicalization
2) Recompute hash over canonical bytes SHA-256 digest recomputation
3) Validate signature against public key reference Origin authenticity + tamper evidence
4) Confirm policy alignment (ID/version/hash) Scope boundary confirmation

Successful verification confirms integrity and provenance under policy scope. It does not confirm output correctness, legality, or compliance conclusions.
